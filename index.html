DROP THE MIKE

Complete guide to splitting audio files, transcribing with AI, and processing transcripts into professional, structured documents.

DROP THE MIKE is a desktop tool that splits long audio and video files into smaller parts, making them easy to upload to Google Gemini for transcription.

After transcription, you work with Mike — an AI agent built on Gemini — that transforms raw conversation transcripts into clean, structured, professional documents.

The process works in 3 simple stages: Transcribe → Process → Generate document.

פיצול קבצי אודיו Splitting Audio Files

ברגע שהתוכנה פתוחה:

1בחרו קובץ

לחצו על "בחר קובץ" ובחרו את קובץ האודיו או הווידאו שלכם (MP3, WAV, M4A, MP4, AVI, MKV ועוד).

2בחרו מספר חלקים

בחרו לכמה חלקים לפצל את הקובץ (2-10+). ככל שהחלקים קטנים יותר, Gemini מתמלל טוב יותר.

3לחצו "פצל קובץ"

הקבצים יישמרו בתיקייה חדשה עם שם הקובץ. אם Gemini אומר שהקבצים ארוכים מדי — לחצו על "קבצים ארוכים מדי? פצל ליותר חלקים".

טיפ: חשבון Gemini בתשלום תומך בקבצים ארוכים יותר. אם אתם משתמשים בחשבון חינמי, פצלו ליותר חלקים.

Once the program is open:

1Select a file

Click "Select File" and choose your audio or video file (MP3, WAV, M4A, MP4, AVI, MKV and more).

2Choose number of parts

Choose how many parts to split into (2-10+). Smaller parts = better transcription quality in Gemini.

3Click "Split File"

Files are saved in a new folder named after your file. If Gemini says files are too long — click "Files too long? Split into more parts".

Tip: A paid Gemini account supports longer audio files. If using a free account, split into smaller parts.

שלב 1 — תמלול עם מייק Stage 1 — Transcription with Mike

שיחה ראשונה ב-Gemini

בשלב הזה מייק מאזין לקבצי ה-MP3 ומתמלל אותם לטקסט.

1פתחו שיחה חדשה עם מייק

היכנסו לסוכן מייק ופתחו שיחה חדשה:

פתחו שיחה עם מייק

2העלו קובץ MP3 וביקשו תמלול

העלו את קובץ ה-MP3 הראשון לחלון השיחה ואמרו למייק: "תמלל בעברית".

מייק יאזין לקובץ ויפיק תמלול מלא ומדויק עם חלוקה לדוברים.

העלאת קובץ MP3 לשיחה עם מייק — מייק נכנס למצב תמלול

3שמרו את התמלול

כשמייק סיים לתמלל — העתיקו את כל התמלול ושמרו אותו בקובץ טקסט במחשב. תנו לקובץ שם לפי מספר החלק (למשל: 1.txt, 2.txt וכו').

4המשיכו עם שאר הקבצים

באותו חלון שיחה, העלו את קובץ ה-MP3 הבא וביקשו תמלול. חזרו על התהליך לכל הקבצים.

חשוב: אם שמתם לב לירידה באיכות התמלול (טעויות, החמצות) — פתחו חלון שיחה חדש עם מייק והמשיכו לתמלל שם. זה קורה כי לפעמים חלון השיחה "מתעייף" אחרי קבצים רבים.

First Gemini conversation

In this stage, Mike listens to your MP3 files and transcribes them to text.

1Open a new chat with Mike

Go to the Mike agent and start a new conversation:

Open a chat with Mike

2Upload an MP3 file and request transcription

Upload your first MP3 file to the chat and tell Mike: "Transcribe in Hebrew" (or English, depending on the language).

Mike will listen to the file and produce a full, accurate transcription with speaker identification.

Uploading an MP3 file to Mike — Mike enters transcription mode

3Save the transcription

When Mike finishes — copy the entire transcription and save it as a text file on your computer. Name the file by part number (e.g., 1.txt, 2.txt, etc.).

4Continue with remaining files

In the same chat window, upload the next MP3 file and request transcription. Repeat for all files.

Important: If you notice transcription quality dropping (errors, missed words) — open a new chat window with Mike and continue transcribing there. This happens because the chat window can get "tired" after many files.

שלב 2 — עיבוד התמלול Stage 2 — Processing the Transcript

שיחה שנייה ב-Gemini

בשלב הזה מייק מנתח את כל התמלול ומכין תכנית עבודה.

1פתחו שיחה חדשה עם מייק

חשוב: פתחו חלון שיחה חדש (לא אותו חלון של התמלול!).

פתחו שיחה חדשה עם מייק

2העלו את כל קבצי התמלול

העלו את כל קבצי הטקסט שיצרתם בשלב הקודם (1.txt, 2.txt, 3.txt וכו') לתוך חלון השיחה ביחד.

העלאת כל קבצי התמלול לשיחה חדשה — מייק נכנס למצב עיבוד

3מייק ינתח וישאל שאלות

מייק יזהה אוטומטית שהוא עכשיו במצב עיבוד תמלול. הוא:

  • יזהה את הדוברים ויבקש אישור
  • ימפה את הנושאים שנדונו בשיחה
  • ישאל שאלות הבהרה לגבי מה אתם צריכים מהתמלול
  • יציע מבנה למסמך הסופי

ענו על השאלות שלו — זה עוזר לו להכין מסמך מדויק יותר.

4קבלו את קובץ ה-ID

בסוף השלב, מייק ייצור קובץ ID — "תעודת זהות" של התמלול. הקובץ הזה מכיל את כל המידע שמייק צריך כדי להפיק את המסמך הסופי: מי הדוברים, מה הנושאים, ומה המבנה שאישרתם.

הורידו את קובץ ה-ID ושמרו אותו במחשב.

Second Gemini conversation

In this stage, Mike analyzes the entire transcript and prepares a work plan.

1Open a new chat with Mike

Important: Open a new chat window (not the same one used for transcription!).

Open a new chat with Mike

2Upload all transcription files

Upload all text files you created in the previous stage (1.txt, 2.txt, 3.txt, etc.) into the chat window together.

Uploading all transcript files to a new chat — Mike enters processing mode

3Mike will analyze and ask questions

Mike will automatically detect he's in transcript processing mode. He will:

  • Identify speakers and ask for confirmation
  • Map the topics discussed in the conversation
  • Ask clarifying questions about what you need from the transcript
  • Suggest a structure for the final document

Answer his questions — this helps him produce a more accurate document.

4Receive the ID file

At the end of this stage, Mike creates an ID file — an "identity card" for the transcript. This file contains everything Mike needs to produce the final document: who the speakers are, what topics were covered, and the structure you approved.

Download the ID file and save it on your computer.

שלב 3 — יצירת המסמך הסופי Stage 3 — Generating the Final Document

שיחה שלישית ב-Gemini

בשלב הזה מייק מייצר את המסמך המוגמר אוטומטית.

1פתחו שיחה חדשה עם מייק

שוב — פתחו חלון שיחה חדש (שלישי!).

פתחו שיחה חדשה עם מייק

2העלו את התמלול + קובץ ה-ID

העלו ביחד את כל קבצי התמלול (1.txt, 2.txt...) ואת קובץ ה-ID שקיבלתם בשלב הקודם.

מייק יזהה את קובץ ה-ID אוטומטית וידע בדיוק מה לעשות.

3קבלו את המסמך!

מייק ייצר את המסמך הסופי לפי המבנה שאישרתם — בלי שאלות, בלי עיכובים. פשוט מסמך מסודר ומקצועי.

טיפ: אחרי שקיבלתם את המסמך, אפשר להדביק אותו ב-עורך Markdown עברי — כלי חינמי ליצוא PDF מקצועי עם תמיכה מלאה בעברית.

Third Gemini conversation

In this stage, Mike generates the finished document automatically.

1Open a new chat with Mike

Again — open a new chat window (third one!).

Open a new chat with Mike

2Upload transcripts + ID file

Upload together all transcript files (1.txt, 2.txt...) and the ID file you received in the previous stage.

Mike will automatically detect the ID file and know exactly what to do.

3Receive your document!

Mike will generate the final document according to the structure you approved — no questions, no delays. Just a clean, professional document.

Tip: After receiving the document, you can paste it into the Hebrew Markdown Editor — a free tool for professional PDF export with full Hebrew support.
